ReBarUEFI is an open-source UEFI DXE driver that enables Resizable BAR (ReBAR) on systems that lack official firmware support. By injecting a lightweight driver into your UEFI firmware's DXE volume, ReBarUEFI configures PCIe Base Address Register sizing during boot — delivering the same GPU performance improvements officially supported systems enjoy.

## What is Resizable BAR?

Resizable BAR (ReBAR) is a PCIe feature that allows the CPU to access the entire GPU framebuffer at once instead of in 256 MB chunks. Most modern motherboards support it natively, but many older systems with PCIe-capable CPUs and GPUs are artificially locked out by their firmware.

ReBarUEFI removes that limitation by hooking into the UEFI PCI enumeration process and configuring BAR sizes according to your settings — with no permanent BIOS modification required beyond injecting the driver module.

<Note>
  Intel Arc GPUs **require** Resizable BAR to function optimally. ReBarUEFI is an effective solution for older platforms that pair an Arc GPU with a motherboard that predates official ReBAR support.
</Note>

## How it works

<Steps>
  <Step title="Build or download the FFS module">
    Get the prebuilt `ReBarDxe.ffs` from the [GitHub Releases](https://github.com/xCuri0/ReBarUEFI/releases) page, or build it yourself from source using the provided EDK2 build script.
  </Step>
  <Step title="Inject the driver into your UEFI firmware">
    Use UEFITool to open your motherboard's firmware image and add the FFS module to the DXE volume. Apply any needed UEFIPatch fixes for your platform.
  </Step>
  <Step title="Flash the modified firmware">
    Flash the patched firmware to your motherboard using the vendor's update tool or a hardware programmer. Enable 4G Decoding and disable CSM in BIOS settings.
  </Step>
  <Step title="Configure the BAR size with ReBarState">
    Run the `ReBarState` utility (Windows or Linux) to set your desired maximum BAR size. Value `32` enables unlimited BAR sizing for full performance.
  </Step>
</Steps>

## Key features

<CardGroup cols={2}>
  <Card title="Broad compatibility" icon="server">
    Works on Sandy Bridge through modern platforms. Tested on X99, Z170, Z270, Z370, and many others.
  </Card>
  <Card title="Safe by design" icon="shield-check">
    Detects CMOS resets and automatically disables ReBAR if the system clock is cleared, preventing boot failures.
  </Card>
  <Card title="Windows and Linux support" icon="display">
    The ReBarState configuration utility runs on both Windows (requires admin) and Linux (requires root).
  </Card>
  <Card title="UEFI patch library" icon="puzzle-piece">
    Bundled UEFIPatch patterns fix BAR size limits, 64-bit BAR downgrades, and MMIO space constraints in older firmware.
  </Card>
</CardGroup>

<Tip>
  Start with a smaller BAR size (e.g., value `8` = 256 MB) to verify stability before setting `32` for unlimited sizing. This is especially recommended for systems with older firmware that may not handle large BARs correctly.
</Tip>
